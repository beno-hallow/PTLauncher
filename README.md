# PT Launcher

A tiny always-on-top floating launcher that sits over your Pro Tools edit
window. A compact strip of buttons that can open apps **and** fire custom Pro
Tools actions — run an AudioSuite plugin on the selected clip, open RX Connect
and send to RX, run keystrokes, or run your own AppleScripts. The panel only
appears while Pro Tools is the active app, and a waveform icon in the menu bar
toggles it.

## Documentation

- **Installing & using it** → [INSTALL.md](INSTALL.md)
  Download the signed app, grant permissions once, customize buttons. Updates
  are automatic (Sparkle).
- **Cutting & publishing releases** (maintainers) → [RELEASING.md](RELEASING.md)
  One-time Sparkle/Developer ID setup, then `./release.sh` per version.

## Repo layout

| File | What it is |
|------|------------|
| `main.swift` | The whole app (AppKit, single file). Sparkle is compiled in only when present. |
| `dev.sh` | Fast local dev loop: compile + run (no Sparkle, no signing). `./dev.sh watch` to rebuild on save. |
| `setup-sparkle.sh` | One-time: download Sparkle, generate the update-signing key. |
| `build.sh` | Build + Developer ID sign (hardened runtime) with Sparkle embedded. |
| `release.sh` | Build → notarize → staple → sign appcast → publish to GitHub Releases. |
| `VERSION` | Current version string, stamped into the build. |
| `appcast.xml` | Sparkle update feed (regenerated by `release.sh`). |
| `sparkle_pubkey.txt` | Public update-signing key, baked into the app. Safe to commit. |

## Requirements

macOS 11+. To build: Apple command-line tools (`xcode-select --install`).
To release: a paid Apple Developer account (Developer ID + notarization).
